# 定义
复利机器是指：企业具有绝对的优势，能在未来10-20年，能保持当前的业绩，甚至获得更好的业绩。其分类一般有：
  - 利润投入再生产型：利润投入生产，滚雪球，飞轮效应。
  - 难以扩大规模，利润投入生产，公司可能不会有过去那样的回报率，但公司回购注销、股东得到分红再买入，此时股东所占权益仍有复利效应。
复利机器的入选应当是苛刻的，否则应当归类为“难以看到复利效应”。

# 分析与判断思路
1. 要说出这家公司真正在做什么，供应商是谁，客户是谁，再下游的客户是谁（没有的话就不写）？它靠什么赚钱、怕什么，全在这一句里。
2. 优势分析
 - 供给侧竞争优势：企业拥有竞争者或新进入者无法复制的低成本结构（包括但不限于优质矿、低价能源、技术专利、流程管理），企业可以通过提供包括但不限于更低价格、更频繁广告、更好服务等进行打击对手。但要警惕某些低成本来源，如技术专利、流程管理等，因为这些很可能被对手绕开、借鉴、抄袭。
 - 需求侧竞争优势：企业能利用以下因素，形成差异化，并锁定客户：
    - 习惯：口味、操作习惯（如老年智能手机）。
    - 转换成本：客户需要花费大量的时间、金钱和精力才能更换供应商，如你的软件必须与别人兼容。
    - 搜寻成本：当寻找可接受的替代品或服务代价高昂时，客户就被绑定在现有的供应商上了。
 - 规模经济效益竞争优势：
    - 如果固定成本在总成本中占比较高，那么单位成本就会随着产量的上升而下降。这样即使技术上并无差别，在位企业也可以通过大规模生产而享受相较于竞争对手的低成本优势。
    - 规模经济效益体现在价值端：规模越大，整体价值越高，“1+1>2”，例如士兵人数变为2倍，部队战斗力变为4倍。
 - 事实上，真正持久的竞争优势来自供给侧竞争优势与需求侧竞争优势的相互作用，即规模经济效益与客户锁定的有机结合。
 - 网络效应有时会同时加强客户锁定和规模经济效应，网络效应也有强弱之分（多边网络效应强于双边网络效应）。
 - 把ROE（净资产收益率）通常大于15%作为参考，但不绝对。
3. 如何判断企业是利润投入再生产型的成长飞轮，还是利润稳定的现金机器，还是处处漏水的“难以看到复利效应”（这一点很难，不强求回答，但尽量去想）
 - TAM（潜在市场总量）有多大，这个市场已经被满足了吗？
 - 产品在市场上是平庸，还是有独特优势？
 - 存在用户越多-产品越有优势，产品越有优势-用户越多的效应吗（如网络效应）。 
4. 如果企业有多项业务，应该分开判断各个业务是否具有竞争优势，然后看各个业务之间有无协调效应。
5. 企业文化（包括纠错能力、执行能力）：历史来看，企业打过哪些硬仗，结果如何？商业模式经过了如何的迭代？从ROE/净利润看，企业迭代后是不是上了一个层次？我不要一句企业文化是xxx，员工认同度高这种空话，我要具体支撑案例。
6. 近三年有出现业绩下滑吗，因为什么，怎么应对的？
7. 当下和可预见的未来有什么隐忧，公司提示了哪些风险，未来两年内有明确会稀释股东权益的事发生吗。


# 信息来源：
  - 数据类：财报内的数据可直接采信。其它任何单一渠道获取的数据不得直接采信，必须通过另一独立渠道交叉验证后方可视为可靠，验证采用容差比对而非精确匹配——因数量级取舍、四舍五入造成的微小差异可接受，不要求数字绝对相等。例如：A 源为 0.1 亿、B 源为 1001 万，视为相等；又如 1.5 与 1.49 视为相等。
  - 文字类：文字材料不作交叉验证的要求。
  - 分析材料的来源，优先级从高到低分别是：财报，官方声明，新闻报道，自媒体。

# 数据获取流程

## 当用户请求分析某只个股时，必须**先收集材料、再动笔**。

## 财报类：是最关键的一类材料，按下面的 SOP 拉取；详细命令模板与市场映射见 `references/download_protocol.md`。

### 1. 解析意图
- 从用户的话里抽出：**公司名** + **报告类型**（默认年报）+ **年份范围**（默认近 3 年）。
- 公司名过于含糊、无法定位到唯一一家上市公司时，**反问用户要准确代码**，不要瞎猜。

### 2. 公司名 → 规范 ticker
- 用你的知识把中英文公司名翻成规范股票代码；三市场形态与示例见 `references/download_protocol.md` §2 表格。
- **只输出规范代码本身，不带 `.HK` / `.SH` / `.SZ` 等后缀**——下游下载器根据代码形态自己判市场。
- 拿不准就反问用户要代码，**禁止凭印象编造**。

### 3. 默认下载范围
- 近 3 年年报（必备）
- 最新一期半年报（A/HK 生效；美股无半年报，自动忽略）
- 最新一期季报（A/US 生效；港股可能没有季报，自动跳过，不算失败）

用户在对话里另行指定年份/类型时，按用户指定为准。

### 4. 先查本地，再下载
- 下载前先检查输出目录 `<workspace-root>/filings/<ticker>/` 是否已有目标年份的报告。
- 已有 → 直接读取，**不重复下载**。
- 没有 → 调 `cola-fetch` 下载。
- cola-fetch 内部的幂等机制（已有则跳过）见 `references/download_protocol.md` §5。

### 5. 调用 cola_fetch
- 命令模板、venv 建立、参数全表见 `references/download_protocol.md` §1；三市场调用示例见 §6。

- 失败时**重试 1 次**。
- 仍失败 → 在报告里如实标注"未能获取 X 年财报"，按写作规则第 1 条"查不到直接标注，不硬撑"，继续推进其余分析，**不要因某一期财报下不到就整份报告不出**。

### 5.1 SEC 限流应对（仅美股）
- 被 SEC 限流（HTTP 429/503，退出码 `3`）时，**反问用户**要真实 User-Agent（含联系邮箱），用 `--sec-user-agent` 参数重跑。
- 默认 UA、退避策略、退出码语义详见 `references/download_protocol.md` §8.1。

### 6. 读取分析
- 读取落盘报告后，按既有「写作规则」+「输出要求」产出报告。
- 落盘路径与命名规则（含美股 `.htm` / 港股A股 `.pdf` 区分）见 `references/download_protocol.md` §4。

### 交叉验证如何与下载协同
- **财报正文（定性叙述 + 财务报表）**：以 `cola-fetch` 下载的**官方源 PDF** 为主源——披露易（港股）/ 巨潮（A股）/ SEC（美股）本就是官方一手源，**不需要再下一份同公司财报来做"第二渠道"**。

## 非财报类
- 非财报类的信息也是重要的参考材料，主要信息可以是：
  - 公司的报道及管理层言论（包括但不限于业务布局、项目进展、未来规划）
  - 第三方评论（包括但不限于比较评论、历史复盘）
- **数值类交叉验证**：用**联网检索**（新闻 / 财经站点 / FMP）对关键数字做容差比对，符合「信息来源」第 1 条的容差规则。
- **文字类材料**（公告、新闻）不强制交叉验证。
- **禁止**反复下载同一份同公司财报充当"独立渠道"——那会陷入死循环。

# 写作规则
  1. 只用真实信息。查不到的直接标注，不硬撑，不编。
  2. 敢下判断。「既可能好也可能坏」这种话一句不许有。
  3. 每个判断带证据：数据、引用、具体事实。在每个判断后面用方括号“[]”列出引用，格式：[xxx年报xxx节]（如：[FY2025，主要会计数据章节]），or，[[网站](网址)]（如：[[新浪新闻](https://k.sina.com.cn/article_3266943013_mc2b9982503301lbi6.html?from=news)）]
  5. 这些词禁用：赛道很大、团队优秀、前景广阔、蓝海市场。
  6. 篇幅以说透为准。两千字说得清就两千字，要七千字就七千字。
  7. 不输出情绪化或安抚性措辞，不输出目标价、估值区间、仓位建议、买卖建议。
  8. 中文撰写。

# 输出要求
- 格式：markdown文档
- 目录：将生成的报告保存到**当前项目根目录**。
- 命名按 denote：`企业名称-YYYYMMDD.md`，如 `腾讯控股-20260216.md`
